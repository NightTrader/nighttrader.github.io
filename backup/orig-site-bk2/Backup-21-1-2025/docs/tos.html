1. Description of Services#back to top

1.1 We provide an efficient, convenient and secure site for storing, buying and selling digital assets. Users deposit their digital assets to the NightTrader Multi Signature Wallet and decentralized Exchange, and, through the site, authorizes transactions to move those assets or buy and sell those assets for other assets.

1.2 The decentralized nodes that host the order books charge a hosting fee for both parties involved in an active exchange additionally to a minimal transaction fee to cover the mining/staking the expense of withdrawals to the respective blockchain.

1.3 NightTrader is a non-custodial wallet when storing digital assets. This means we do not own, control, or take custody of any digital assets held by Users on the site. When funds are placed into the order books to be sold or bought, the funds are issued by the user via a blank check which is later routed by a decentralized consensus of nodes which use a type of shamir's secret sharing and threshold key(a giant multisignature represented by single keys);

1.4 NightTrader’s order books are semi-custodial. When an order is placed, the User signs a raw transaction with 1 of 2 signatures. This allows the decentralized exchange backend to complete the trade at a later time while the user is offline. It allows this backend to also update the payee many times without broadcasting and promptly clear debts to mitigate any risks of collusion.

1.5 All the assets are kept on 2 of 2 multi signature addresses. To perform any operations with assets, it is required to have the users signature and the exchange signature. NightTrader can not move your assets until you sign your part of the transaction. NightTrader does not keep private keys of user accounts. The private key of a user account is based on a signature password that only the user knows. This can be verified by looking at our open source website and front end. NAOME is not an owner of the exchange, it is an owner of the domain and project director. The domain redirects to github to which NAOME contracts coders to update this community driven project. The hosting fee is therefore shared between users, the developers and NAOME who helps bring it to market. There is no ICO, no security and no custody.

1.6 NightTrader can not restore the signature password if it gets lost or forgotten. You agree to keep the signature password in a safe place otherwise your funds can not be accessed.

1.7 Because all exchange operations are bound with transaction outputs it is a requirement that your operations have to be greater than 0.00001 BTC (10k Satoshis), otherwise fees for sending funds or debts clearance will increase.

1.8 Force majeure (for Bitcoin based blockchains that support checklocktimeverify and Ethereum based coins only otherwise please carefully check the manual backup method): In case of a deny-of-service attack(DDOS) and when the exchange is forced to stop operations each user can retrieve their funds via a the time locked script within a 1 year lock time, so in cases of force majeure the user can withdraw funds after one year. For coins that don’t support checklocktimeverify scripts (such as Monero) users must manually backup their force majeure which can be a signed nlocktime transaction set within 1 year into the future. A manual service would require backups for each trade only after funds clear. We don’t guarantee we will deploy the manual service upon release of the exchange. Users will have to occasionally make a transaction if no account activity has happened for many months which results in a small fee on the blockchain. This is to prevent DDOS against backend nodes to double spend an almost expired timelock.
